# ТЗ: Переработка системы атрибутов (Глобализация атрибутов)

## Контекст
Сейчас атрибуты жестко привязаны к Клиентам. Необходимо превратить «Атрибуты» в универсальный справочник, который можно назначать разным сущностям системы.

## Задача: Рефакторинг раздела "Атрибуты"

### 1. Изменение структуры таблицы
В разделе "Атрибуты" необходимо добавить колонку "Тип (сущность)", чтобы пользователь видел, к чему относится каждый атрибут.
* **Новая структура:** `ID` | `Название` | `Тип (сущность)` | `Действия`

### 2. Форма добавления/редактирования
При создании нового атрибута или редактировании существующего, добавь выпадающий список (select) "Тип сущности" с выбором из вариантов:
* `Клиент`
* `Заказ`
* `Детали (склад)`
* `Поставщики`

### 3. Логика работы (Backend/Frontend)
* **Сохранение:** В базе данных атрибут должен теперь сохраняться с полем `entity_type` (значения: 'client', 'order', 'stock', 'supplier').
* **Применение:**
    * При просмотре/редактировании **Клиента** должны подтягиваться и отображаться только те атрибуты, где `entity_type == 'client'`.
    * При создании **Заказа** — только те, где `entity_type == 'order'`.
    * Аналогично для **Склада** (`stock`) и **Поставщиков** (`supplier`).
* **Миграция данных:** Все текущие (существующие) атрибуты, которые уже добавлены, автоматически присвоить к типу `Клиент`, чтобы ничего не пропало.

### 4. Интерфейс
* В формах добавления/редактирования (Клиента, Заказа и т.д.) теперь нужно добавить блок "Дополнительные атрибуты", который динамически опрашивает справочник атрибутов и выводит только те, что подходят по типу сущности.
