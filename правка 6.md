# ТЗ: Добавление функции ручной архивации заказов по кнопке

## Контекст
В разделе "Заказы" есть вкладка "Архив". В данный момент заказы отправляются туда (или фильтруются для нее) в зависимости от выбранного статуса заказа ("В работе", "Выдан", "Задержка", "Отменен"). 
Пользователю нужно добавить *дополнительную* возможность отправлять заказ в архив по нажатию специальной кнопки, независимо от того, какой у него сейчас статус.

## ВАЖНОЕ ОГРАНИЧЕНИЕ (КРИТИЧНО)
**Нельзя ломать или удалять текущую логику работы добавления в архив через статусы заказа!** Архивация при смене статуса ("В работе", "Выдан", "Задержка", "Отменен") должна работать точно так же, как работает сейчас. Новая кнопка — это исключительно *дополнение*, а не замена существующего механизма.

## Задача: Интеграция ручной архивации
Внеси следующие изменения, аккуратно интегрируя их в текущую архитектуру:

1. **Обновление UI (Кнопка действия):**
   - В списке активных заказов добавь новую кнопку "В архив" (например, иконка коробки 📦). 
   - Размести её либо в колонке действий для каждого заказа, либо сделай массовую кнопку над таблицей, если реализовано выделение заказов чекбоксами.

2. **Синхронизация фильтров (Frontend/Backend):**
   - Проанализируй, как сейчас заказы попадают во вкладку "Архив" (скорее всего, через фильтрацию по статусам).
   - Введи в модель заказа новый параметр/флаг, например, `is_archived_manually` (boolean, по умолчанию False).
   - Обнови фильтр вкладки "Архив": теперь там должны отображаться заказы, у которых ЛИБО статус равен архивному ("Выдан", "Отменен" и т.д.), ЛИБО флаг `is_archived_manually` равен `True`.
   - Обнови фильтр вкладки "Активные": из неё должны скрываться заказы, у которых установлен флаг `is_archived_manually = True`.

3. **Обработчик кнопки:**
   - Нажатие на новую кнопку должно только менять флаг `is_archived_manually` на `True` и убирать заказ из текущего вида. Статус заказа при этом меняться **не должен**.
   - Добавь во вкладке "Архив" обратную кнопку "Разархивировать" (которая снимает флаг `is_archived_manually`), чтобы заказ вернулся в активные, если его статус позволяет там находиться.
