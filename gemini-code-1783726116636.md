# ТЗ: Исправление логики склада, атрибутов и автоматизации заказов

## 1. Синхронизация артикулов (Склад <-> Заказы)
* **Проблема:** Артикул, введенный в заказе, не синхронизируется со складом.
* **Решение:** * Если деталь выбрана из базы склада, артикул должен автоматически подтягиваться в заказ. 
    * Если артикул вводится вручную при создании заказа — при сохранении заказа система должна проверять наличие такого артикула на складе. Если его нет — предложить создать новую позицию на складе или обновить существующую.

## 2. Глобализация атрибутов (Разделы: Заказ, Склад, Поставщики)
* **Проблема:** Поля атрибутов не отображаются в формах создания/редактирования для сущностей, отличных от "Клиент".
* **Решение:**
    * Внедрить динамическую генерацию полей ввода для атрибутов в зависимости от `entity_type` (Заказ, Склад, Поставщики).
    * При открытии окна редактирования/создания сущности система должна делать запрос к списку атрибутов, отфильтрованных по типу, и рендерить соответствующие `input` поля для каждого найденного атрибута.

## 3. Контроль остатков при добавлении в заказ
* **Логика:** Запретить добавление позиции в заказ, если остаток на складе <= 1 (при включенной настройке).
* **Настройки:** В меню "Настройки" добавить чекбокс "Блокировка заказа при отсутствии товара" (По умолчанию: ВКЛ).
* **Обработка ошибки:** При попытке добавить товар, которого нет в наличии, выводить модальное окно:
    * Текст: "Данной позиции недостаточно на складе".
    * Кнопки: 
        1. "Изменить количество" (возврат к выбору).
        2. "Добавить позицию" (принудительное добавление, игнорирующее остаток).

## 4. Финансовая логика и автоматизация полей
* **Поле "Итого цена продажи":** * Должно стать **недоступным для ручного ввода**.
    * Оно должно пересчитываться автоматически (сумма цен всех позиций в заказе).
* **Поле "Сумма долга":**
    * Сделать **только для чтения** (как текст или статичное число).
    * **Видимость:**
        * Статус "Не оплачен" или "Оплачен": Поле "Сумма долга" **полностью скрыто**.
        * Статус "Частично оплачен": Поле "Сумма долга" **видимо** (расчет: `Итого цена` - `Предоплата`).
    * **Автоматизация:** При любом изменении состава заказа (добавление/удаление позиции) или изменении суммы предоплаты, значения должны мгновенно пересчитываться через JS без перезагрузки страницы.
