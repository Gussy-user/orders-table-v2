# ТЗ: Исправление логики частичной оплаты заказа

## Контекст
В текущей форме заказа избыточное количество финансовых полей, что путает пользователя. Необходимо привести интерфейс и логику к простому виду.

## Технические требования:

### 1. Очистка полей
- Удалить из формы все лишние поля: "Оплачено", "Осталось" и "Долг" (если они дублируют смысл).
- Оставить в форме строго следующие поля:
    1. **Итого цена продажи (₽)** — вводится пользователем.
    2. **Статус оплаты** — выпадающий список: ["Не оплачен", "Частично оплачен", "Оплачен"].
    3. **Внесенная предоплата (₽)** — поле должно быть скрыто, если статус оплаты НЕ "Частично оплачен".

### 2. Динамическая логика (JS)
- Реализовать условие:
    - Поле **"Внесенная предоплата (₽)"** появляется (становится видимым) только при выборе статуса оплаты **"Частично оплачен"**. Если выбран любой другой статус, поле скрывается.
- Автоматический расчет долга:
    - Добавить информационное поле (только для чтения) **"Сумма долга (₽)"**.
    - Логика расчета:
        - Если статус "Не оплачен": Сумма долга = Итого цена продажи.
        - Если статус "Частично оплачен": Сумма долга = Итого цена продажи - Внесенная предоплата.
        - Если статус "Оплачен": Сумма долга = 0.
    - Пересчет должен происходить автоматически при изменении любого из двух полей: "Итого цена продажи" или "Внесенная предоплата".

### 3. Сохранение данных
- В базу данных заказа должны сохраняться только: "Итого цена продажи", "Статус оплаты" и "Внесенная предоплата".
- "Сумма долга" является производным полем (вычисляемым) и не требует отдельного жесткого сохранения в БД, если она всегда пересчитывается при открытии заказа.
