# ТЗ: Учет остатков и автоматическое списание товаров

## Задача 1: Добавление колонки "Количество" на склад
**Проблема:** В таблице склада сейчас отсутствует информация о текущем остатке товара.
**Решение:**
1. В форму добавления/редактирования товара на складе добавить поле "Количество" (тип: integer, целое число).
2. В таблицу "Склад" добавить новую колонку "Количество" между "Артикул" и "Закупка".
3. Обновить модель данных (БД) склада, добавив поле `quantity` (текущий остаток).

## Задача 2: Автоматическое списание товара при оплате заказа
**Проблема:** Товар не списывается со склада автоматически после завершения сделки.
**Решение:**
1. **Логика списания:** При смене статуса заказа на "Выдан" или "Оплачен" (или при сохранении заказа, если статус уже такой), система должна автоматически найти позиции в этом заказе на Складе и вычесть их количество из поля `quantity`.
   - Формула: `Новый остаток = Текущий остаток - Количество в заказе`.
2. **Настройки (Settings):**
   - Добавить в настройки программы переключатель (checkbox): "Автоматическое списание товара при оплате".
   - Состояние по умолчанию: `Включено`.
3. **Безопасность:**
   - Если при попытке списания товара на складе недостаточно (количество в заказе > остатка на складе), система должна выдать предупреждение пользователю, но позволить завершить заказ (либо заблокировать, если остаток < 0 — уточни у пользователя).
