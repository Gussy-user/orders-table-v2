## Задача 2: Принудительный запуск интерфейса в Google Chrome из .exe
**Проблема:** Приложение запускается через скомпилированный .exe файл (бэкенд), который автоматически открывает UI в браузере по умолчанию (в Windows 10 это Edge). Пользователю необходимо, чтобы интерфейс всегда открывался в Google Chrome.
**Решение:**
В коде бэкенда (там, где вызывается открытие окна браузера после старта локального сервера) нужно изменить логику запуска:
1. Задать поиск Google Chrome по стандартным путям Windows (например, `C:/Program Files/Google/Chrome/Application/chrome.exe %s` или использовать вызов консольной команды `start chrome <url>` через модуль `subprocess`).
2. Попытаться принудительно открыть локальный URL именно через найденный исполняемый файл Chrome.
3. Обязательно реализовать fallback (запасной вариант): обернуть вызов в `try/except`. Если Chrome не найден на компьютере (или установлен в нестандартную папку), программа не должна крашиться — она должна просто открыть интерфейс в браузере по умолчанию (стандартный `webbrowser.open(url)`).
