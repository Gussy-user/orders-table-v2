# Спецификация: Добавление блока создания авто в форму "Новый клиент"

## Контекст
В данный момент функционал добавления автомобиля доступен только в режиме редактирования карточки клиента. Необходимо расширить форму "Нового клиента", чтобы можно было создать клиента и сразу привязать к нему автомобиль в едином окне.

## Технические требования:

### 1. Интерфейс (UI)
- В форму "Добавление клиента" необходимо внедрить блок "Данные автомобиля".
- Этот блок должен полностью дублировать поля, которые сейчас используются при добавлении авто в режиме редактирования клиента (Марка, Модель, VIN-код и т.д.).
- Чтобы не перегружать интерфейс, этот блок можно разместить под основными данными клиента (ФИО, телефон).

### 2. Логика сохранения (Backend/Database)
- При нажатии кнопки "Сохранить клиента":
    a. Система должна сначала создать запись нового клиента в базе данных.
    b. Если в блоке "Данные автомобиля" были заполнены поля, система должна автоматически создать запись об автомобиле и привязать её к ID только что созданного клиента.
- Вся транзакция должна происходить атомарно (или клиент, или клиент с привязанным авто).

### 3. Пользовательский сценарий
- Пользователь открывает страницу создания нового клиента.
- Заполняет данные ФИО.
- Заполняет данные авто (Марка, VIN).
- Нажимает "Создать".
- В результате в базе появляется клиент, и в его профиле уже отображается созданный автомобиль.
