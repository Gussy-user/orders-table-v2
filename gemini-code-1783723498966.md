# Спецификация: Интерактивная форма выбора мессенджеров

## Задача: Динамическое отображение полей ввода для связи
В форме добавления/редактирования клиента необходимо реализовать логику, при которой поле ввода появляется только при выборе соответствующего мессенджера.

### Технические требования:
1. **Интерфейс (UI):**
   - Добавить блок "Каналы связи".
   - Внутри разместить 3 чекбокса: `Telegram`, `WhatsApp`, `MAX`.
   - Под каждым чекбоксом (или рядом) изначально должно быть скрыто текстовое поле (`<input type="text">`) с плейсхолдером "Введите ник или номер".

2. **Логика поведения (JS):**
   - Поле ввода для конкретного мессенджера должно появляться ТОЛЬКО в момент нажатия на чекбокс этого мессенджера.
   - Если пользователь снимает галочку, соответствующее поле ввода должно снова скрываться (и желательно очищаться).
   - Логика должна позволять пользователю выбрать несколько мессенджеров одновременно (например, Telegram и WhatsApp), и для каждого из них должно быть доступно свое поле ввода.

3. **Сохранение данных:**
   - Данные из каждого появившегося поля должны привязываться к выбранному типу мессенджера.
   - В БД клиента структура должна позволять хранить объект или массив вида: `{"Telegram": "@user", "WhatsApp": "+7999..."}`.
   - В карточке просмотра клиента отображать только те мессенджеры, которые были выбраны и заполнены.
